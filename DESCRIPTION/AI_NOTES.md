# AI Usage Notes

AI tools were used during the development of this project as learning and debugging assistants.

## How AI Was Used

AI was primarily used for:

* Discussing design decisions
* Finding untested edge cases
* Reviewing logic
* Debugging unexpected behavior
* Explaining concepts related to stacks and parsing

Examples:

* Testing unusual expressions
* Identifying missing validation cases
* Suggesting additional scenarios to verify
* Explaining why certain parser states failed

## How AI Was NOT Used

AI did not generate the final project implementation.

The calculator logic, parsing flow, stack handling, debugging process, and feature implementation were written and integrated manually.

When AI suggested possible issues, those suggestions were independently tested, verified, and implemented only when appropriate.

## Development Approach

The project followed a mentor-style workflow:

1. Build a feature.
2. Test it manually.
3. Discuss failures and edge cases.
4. Improve the implementation.
5. Repeat.

This approach helped strengthen problem-solving skills while still benefiting from AI-assisted review.

## Key Takeaway

AI was used as a technical mentor and reviewer rather than as an automatic code generator.

The goal was to understand the reasoning behind every change instead of simply copying solutions.
